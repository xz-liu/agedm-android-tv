# AGE DM TV

面向 Google TV / Android TV 的 AGE DM 原生客户端。

## 下载

| 版本 | 链接 |
|------|------|
| Release（推荐） | [agedm-tv-release.apk](release-assets/agedm-tv-release.apk) |
| Debug | [agedm-tv-debug.apk](release-assets/agedm-tv-debug.apk) |

最低系统版本：Android 8.0

## 功能

### 浏览与搜索
- 扫码投送、首页、目录、推荐、更新、排行、记录七个分区，遥控器方向键直接切换
- 首页提供今日更新、最近更新、每日推荐与每日追番分区
- 目录支持按地区、版本、类型、年份、排序筛选；排行榜支持按年份筛选
- 全局搜索，支持关键词模糊匹配
- 针对电视遥控器优化焦点逻辑，边界不会异常跳回导航栏

### 播放器
- 左右方向键快进 / 快退 10 秒
- **UP 键一键跳过片头 88 秒**（动画 OP 通常约 90 秒，屏幕右上角短暂弹出提示）
- **OK 键单次暂停 / 播放**（控制栏隐藏时直接切换，无需二次确认）
- 调速：1× / 1.25× / 1.5× / 2×，倍速设置全局记忆
- 多播放源切换，某个源解析或播放失败时自动尝试后续源
- 自动播放下一集时沿用当前成功播放的源
- 记住上次使用的播放源、集数和播放进度，下次进入可继续播放

### 观看记录
- 自动记录每部动画的观看进度（集数 + 时间点），每 5 秒持久化一次
- 记录分区展示最近观看与续播历史
- 详情页可直接从上次播放位置继续观看

### 手机扫码投送
- 扫码页作为主界面标签页直接展示，手机与电视连接同一 Wi-Fi 后即可使用
- 手机端既可以搜索动画名称，也可以提交 AGE 动漫详情页、播放页链接或动画 ID
- 手机连续选择多个动画时，电视会及时切换到最新投送内容
- 搜索结果以动画封面卡片横排展示，支持遥控器左右滚动选择

### 详情页
- 展示动画简介、播放源、选集、系列作品与相关推荐
- 相关推荐对齐桌面站数据，支持显示更多推荐项

## 技术设计

### 导航栏指示器动画

底部导航栏下方有一条 3dp 圆角青色小条，始终跟随当前激活的分区。切换分区时小条以 260ms `DecelerateInterpolator` 平滑滑动，与页面内容的切换动画时长一致，视觉上形成联动。

实现方式：通过 `ViewGroupOverlay` 将指示器 View 添加到 `LinearLayout` 的覆盖层，避免改动导航栏布局结构；位置坐标由 `offsetDescendantRectToMyCoords` 精确换算，动画通过 `ViewPropertyAnimator.x()` 驱动。

### 磁盘缓存（两时间戳设计）

缓存文件头格式：`WRITE_TS\tACCESS_TS\n<数据>`

| 时间戳 | 语义 | 用途 |
|--------|------|------|
| `WRITE_TS` | 数据从网络获取的时刻 | 控制 TTL 新鲜度 |
| `ACCESS_TS` | 数据最后被读取或触碰的时刻 | 控制 14 天冷驱逐 |

- `get(key, ttl)`：命中时更新 `ACCESS_TS`，超过 TTL 则删除
- `peek(key)`：读取数据但不校验 TTL，也不更新时间戳，专供「先展示旧数据」场景
- `touch(key)`：仅刷新 `ACCESS_TS`，用于将某条缓存的 14 天窗口重置
- `evictExpired()`：应用启动时运行，仅驱逐 `ACCESS_TS` 超过 14 天的条目

每次列表请求完成后，会对返回列表中每个动画的 `detail_<id>` 缓存执行 `touch`，确保正在追的番剧（频繁出现在更新列表中）的详情缓存不会被冷驱逐；长期未追的老番则在 14 天无任何访问后自然清理。

> 动画的 metadata 和封面对同一 ID 是永久不变的（换了封面即换了 ID），所以 TTL 设为与 `MAX_AGE_MS`（14 天）相同，实际上只有「14 天内完全没出现过」的缓存才会被删除。

### 渐进式加载（Stale-While-Revalidate）

切换导航分区时不显示 loading 状态，而是：

1. 立即从磁盘读取上一次缓存的数据（`peek`）并渲染，**附带页面切入动画**
2. 180ms 后在后台发起网络请求
3. 网络数据返回后静默更新列表内容，**不重复触发切入动画**

若磁盘无缓存则正常显示 loading 状态，网络数据到达后带动画展示。

关键实现：`launchStaleFirstLoad<T>(peek, fetch, render: (T, animate: Boolean) -> Unit)`，第二次 `render` 调用时传入 `animate = false`，`showSections` / `showGrid` 跳过 `animateContentIn`。

### 投送服务 QR 预生成

`AgeTvApplication.onCreate()` 在后台线程同步完成两件事：
1. 启动本地 NanoHTTPD 服务器，获取局域网访问 URL
2. 使用 ZXing 生成 1000×1000 二维码位图

结果存入 `StateFlow<Bitmap?>`。用户首次打开投送页时直接读取 `.value`，无需等待，也不会重复生成。

### 流解析（WebView 代理）

部分播放源需要执行 JavaScript 才能获得真实流地址。播放器维护一个 1×1 不可见的 `WebView`，加载解析页面后通过以下两种方式捕获流地址：

- **网络拦截**：`WebViewClient.shouldInterceptRequest` 捕获符合媒体 URL 特征的请求（`.m3u8 / .mp4 / .flv / .m4s` 等）
- **JS 轮询**：每 500ms 执行一次 JavaScript，读取 `video.currentSrc`、ArtPlayer 内部对象等已知路径，最多重试 24 次（共 12 秒）

两者均在超时前解析成功即停止轮询；超时后回退到 AGE 自身的解析接口。
