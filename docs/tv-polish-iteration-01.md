# TV Polish Iteration 01

日期：2026-04-16

## 目标

这一轮集中处理 TV 可用性问题，重点是原生浏览页、目录页稳定性和播放器体验。

## 问题清单

- [x] 1. 顶部按钮过大，留白偏多，焦点高亮不够明显
- [x] 2. 动画海报过大，希望一屏能看到约 2.5 行
- [x] 3. `目录` 标签页 JSON 异常，并改成和其他标签页一致的海报栅格
- [x] 4. 播放器左下角操作提示常驻，不需要显示
- [x] 5. 倍速改为全局设置，切换动画/分集时沿用同一个倍速
- [x] 6. 浏览记录单独做成标签页，且不必重开 App 才刷新
- [x] 7. 播放器增加更主流的进度条和半透明控件样式

## 已确认根因

### 目录页 JSON 异常

不是筛选参数为空。当前目录请求已经带上了默认值：

```text
/v2/catalog?page=1&size=30&genre=all&label=all&letter=all&order=time&region=all&resource=all&season=all&status=all&year=all
```

真实根因是 AGE 的 `catalog/search` 列表数据里，`play_time` 字段会返回 `null`。当前客户端模型把它定义成非空 `String`，所以会在反序列化时抛异常。

修复方向：

- 列表模型改为可空 / 容错解析
- 目录页只展示海报卡片，不使用“长文本条目式”布局

## 本轮实施方向

- 主界面：缩小 chrome、增强聚焦态、补充历史标签
- 列表：统一缩小卡片尺寸，目录/推荐/更新/搜索/历史都走栅格
- 播放器：移除提示文案、加入进度条、保存全局倍速

## 结果

- 主界面顶部与底部按钮缩小，并加强了焦点态描边
- 海报卡片改小，主界面和栅格页单屏可显示更多内容
- `catalog/search` 已对 `play_time = null` 做容错解析，目录页恢复可用
- 浏览记录拆为独立标签页 `记录`，返回应用或播放器后可即时刷新
- 播放器移除了常驻提示，新增时间条、当前时间/总时长显示，以及半透明按钮样式
- 倍速设置已改为全局持久化，重新进入播放器仍沿用用户上次选择
